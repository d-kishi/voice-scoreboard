# Figma Make 利用ガイド

## 概要

Figma Make は Figma に統合された **AI 駆動のプロトタイピングツール**。
自然言語のプロンプトからデザインを自動生成し、繰り返し修正できる。

本プロジェクトでは **画面レイアウト設計** に使用し、SDD の設計資料に組み込む。

## 基本操作手順

### 1. デザインの新規作成

1. [Figma](https://www.figma.com/) にログイン
2. Drafts（下書き）画面で **Make a design** を選択
3. プロンプト入力欄に作りたい画面の説明を入力
4. 生成されたデザインを確認・修正

### 2. プロンプトの書き方

```
例: バレーボールスコアボード（横画面）。
左チームと右チームのスコアを大きく表示。
背景は暗色、スコアは白文字。
下部に「ロールバック」「リセット」ボタンを配置。
```

**Tips:**
- 具体的なレイアウト指示を含めると精度が上がる
- 「横画面（ランドスケープ）」「暗色テーマ」など制約を明記する
- 1回で完成を目指さず、反復的に修正する

### 3. デザインの修正

- 生成後、プロンプトで追加指示を出して修正可能
- Figma の通常のデザインツールでも手動編集できる
- フレーム・コンポーネント構造は Figma 標準に準拠

## 設計資料への組み込み方法

### Figma ファイル URL の記載

設計書（`.kiro/specs/<feature>/design.md`）に Figma ファイルの URL を記載する。

```markdown
## UI デザイン

- Figma: https://www.figma.com/design/xxxxx/...
```

### スクリーンショットの保存

必要に応じてスクリーンショットを `docs/01_design/images/` に保存する。

> **注意**: cc-sdd の設計書（`.kiro/specs/<feature>/design.md`）はビジュアルに Mermaid 図と外部 URL 参照を使用する。
> ローカル画像は cc-sdd のスコープ外のため、プロジェクト独自の `docs/` 配下で管理する。

```
docs/01_design/
└── images/
    ├── scoreboard-main.png        # メイン画面
    ├── scoreboard-listening.png   # 音声認識中
    └── scoreboard-settings.png    # 設定画面
```

## 本プロジェクトでの活用方針

| 用途 | 詳細 |
|------|------|
| **レイアウト設計** | 画面構成・配置の検討に使用 |
| **プロトタイピング** | SDD 設計フェーズでの UI 参考資料として |
| **コード生成** | 非対応。React Native の実装はすべて手動で行う |

### ワークフローでの位置づけ

```
Figma Make（UI設計）──→ SDD 設計フェーズ ──→ TDD 実装
     ↑                      │
     └── デザイン修正 ←──── フィードバック
```

1. Figma Make で画面レイアウトを作成
2. SDD の `spec-design` フェーズで設計書に URL・スクリーンショットを組み込む
3. `spec-impl` フェーズで設計に基づき React Native コンポーネントを実装

---

## 注意事項

- **React Native コード生成は非対応**: Figma Make が生成するコードは Web 向け（HTML/CSS）のため、本プロジェクトでは使用しない
- **無料プランの制限**: Make の利用回数に制限がある（月間の生成回数上限）
- **デザインは参考資料**: 最終的な UI は React Native の制約に合わせて実装時に調整する
- **Figma Dev Mode**: 無料プランでは制限あり。コードの詳細な inspect が必要な場合は有料プラン要検討
